## 二、各模块详细说明

### 2.1 根目录核心文件

| 文件 | 功能说明 |
|------|---------|
| `main.py` | FastAPI 主入口。注册所有路由（upload, query, fill, files, document_ops），配置 CORS，设置全局异常处理器，初始化数据库，启动/关闭服务器 |
| `config.py` | 全局配置管理。统一从 `.env` 读取所有参数，包括 LLM/VLM/向量模型配置、检索权重、Redis 连接、数据库路径等 |
| `errors.py` | 自定义异常类 `AppError`。统一管理错误码、HTTP 状态码和错误详情，由全局异常处理器统一捕获返回 |
| `requirements.txt` | Python 依赖清单。涵盖后端框架、文档处理、LLM/检索、数据库、工具库和测试框架 |
| `rebuild_index.py` | 索引重建脚本（临时用途）。清理旧数据、重新解析指定 xlsx 文件、更新 SQLite 记录 |
| `start_backend.bat` | Windows 环境下的一键启动脚本 |

### 2.2 API 路由层（`api/`）

#### 2.2.1 `upload.py` - 文件上传与解析

| 接口 | 方法 | 说明 |
|------|------|------|
| `/upload` | POST | 上传文件，支持 MD5 去重（相同文件不重复存储） |
| `/parse` | POST | 提交异步解析任务 |
| `/parse/status/{task_id}` | GET | 查询指定任务的解析状态 |
| `/parse/batch` | POST | 批量提交解析任务 |
| `/template/placeholders` | POST | 分析模板文件中的占位符信息 |

#### 2.2.2 `query.py` - 智能问答

| 接口 | 方法 | 说明 |
|------|------|------|
| `/ask` | POST | 同步问答，返回完整答案及来源 |
| `/ask/stream` | POST | 流式问答（SSE），实时返回答案片段 |
| `/cache/clear` | POST | 清除所有问答缓存 |
| `/cache/clear/{file_id}` | POST | 清除指定文件的问答缓存 |

#### 2.2.3 `fill.py` - 表格回填

| 接口 | 方法 | 说明 |
|------|------|------|
| `/fill` | POST | 表格回填，支持智能模式（LLM 提取）和手动模式（占位符替换） |
| `/download/{file_id}` | GET | 下载已生成的文件 |

#### 2.2.4 `files.py` - 文件管理

| 接口 | 方法 | 说明 |
|------|------|------|
| `/files` | GET | 文件列表查询，支持分页和状态过滤 |
| `/files/{file_id}` | DELETE | 删除指定文件及其关联数据 |
| `/files/{file_id}/reparse` | POST | 重新解析指定文件 |

#### 2.2.5 `document_ops.py` - 文档智能操作

| 接口 | 方法 | 说明 |
|------|------|------|
| `/document/operate` | POST | 自然语言驱动文档操作 |
| `/document/preview` | POST | 预览操作效果（不保存） |
| `/document/extract` | POST | 提取文档内容（支持 PDF/XLSX） |
| `/document/summarize` | POST | 生成文档摘要（支持 PDF） |
| `/document/convert` | POST | 格式转换（docx↔xlsx↔pdf） |
| `/document/merge` | POST | 合并多个文档 |
| `/document/split` | POST | 拆分文档 |
| `/document/batch` | POST | 批量文档操作 |
| `/document/download_result` | GET | 下载操作结果文件 |
| `/document/supported_operations/{file_type}` | GET | 查询指定文件类型支持的操作列表 |

**支持的文件类型与操作**：

| 文件类型 | 支持的操作 | 核心功能 |
|----------|-----------|---------|
| PDF | extract、summarize、extract_page、extract_by_keyword | 按页提取、关键词搜索、大纲获取 |
| DOCX | edit、format、add、delete、extract、summarize | 段落增删改、格式调整 |
| XLSX | edit_cell、format_cell、add_row/delete_row、extract | 单元格操作、行管理 |

### 2.3 业务模块层（`modules/`）

#### 2.3.1 `parser/` - 文档解析（负责人：成员2）

**核心文件**: `document_parser.py`

| 功能 | 说明 |
|------|------|
| PDF 解析 | 使用 VLM 视觉大模型提取文字内容；VLM 不可用时 fallback 到 pdfplumber/Fitz 文本提取 |
| DOCX 解析 | python-docx 提取文字和结构，采用 Markdown 语义分块 |
| XLSX 解析 | pandas 分块解析，每 50 行作为一个 chunk 存入向量数据库 |
| TXT/MD 解析 | 纯文本直接读取，Markdown 语义分块 |
| 实体提取 | LLM 提取文档中的关键实体（人名/地名/机构等）和文档摘要 |
| 文本分块 | Markdown 语义分块策略，保持标题层级和段落完整性 |

#### 2.3.2 `retriever/` - 检索与问答（负责人：成员3）

**核心文件**: `indexer.py` + `hybrid_retriever.py`

##### 索引构建（`indexer.py`）

| 组件/函数 | 说明 |
|----------|------|
| `OllamaEmbedder` | Ollama Embedding API 封装，调用本地部署的嵌入模型 |
| `build_index()` | 主函数：向量索引构建 + BM25 索引构建 + SQLite 元数据写入 |
| `delete_index()` | 删除指定文件的所有向量和 BM25 索引数据 |
| `get_collection()` | 获取 ChromaDB 集合（支持持久化和内存模式） |
| `get_bm25_records()` | 从 SQLite 获取 BM25 原始记录 |
| `_load/_save_bm25_index()` | BM25 索引的持久化加载/保存 |

##### 混合检索（`hybrid_retriever.py`）

| 函数 | 说明 |
|------|------|
| `_hybrid_retrieve()` | 混合检索主函数：向量检索（权重自适应）+ BM25S（jieba 中文分词）+ RRF 倒数排名融合 |
| `rerank_chunks()` | CrossEncoder 重排，基于语义相似度精排 |
| `mmr_diversity_rerank()` | MMR（最大边际相关性）多样性排序，避免结果重复 |
| `detect_conflicts()` | 多源冲突检测：发现不同来源的冲突答案 |
| `fuse_results()` | 冲突融合策略：RRF + LLM 判断 |
| `retrieve_and_answer()` | 检索+问答主入口：检索相关文本 → 构建 Prompt → 调用 LLM 生成答案 |
| `_build_prompt()` | 构建 RAG Prompt，整合检索结果、对话历史、系统指令 |

**检索配置参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `vector_weight` | 0.6 | 向量检索权重 |
| `bm25_weight` | 0.4 | BM25 检索权重 |
| `top_k` | 8 | 返回的 chunk 数量 |
| `reranker_enabled` | True | 启用 CrossEncoder 重排 |
| `reranker_model` | Qwen/Qwen3-Reranker-0.6B | 重排模型 |
| `mmr_enabled` | True | 启用 MMR 多样性排序 |
| `mmr_lambda` | 0.7 | λ 参数（1=完全相关，0=完全多样） |
| `fusion_method` | rrf | RRF 倒数排名融合 |
| `rrf_k` | 60 | RRF 融合超参数 |

## 三、核心数据流

### 3.1 文件上传与解析流程

```
用户 ──▶ POST /upload
              │
              ▼
         检查 MD5 去重
              │
              ▼
         保存文件到 uploads/
              │
              ▼
         解析文档（PDF/DOCX/XLSX/...）
         ├── PDF → VLM 视觉解析（fallback 到文本提取）
         ├── DOCX → python-docx + 语义分块
         └── XLSX → pandas 分块（每50行一个chunk）
              │
              ▼
         构建索引
         ├── 向量索引 → ChromaDB
         └── BM25 索引 → 持久化存储
              │
              ▼
         更新 FileRecord 状态
```

### 3.2 智能问答流程

```
用户 ──▶ POST /ask
              │
              ▼
         检查 Redis 缓存（命中则直接返回）
              │
              ▼
         混合检索
         ├── 向量检索 → ChromaDB（权重 0.6）
         ├── BM25S 检索 → jieba 分词（权重 0.4）
         └── RRF 融合 → 合并两个排序列表
              │
              ▼
         CrossEncoder 重排（可选）
              │
              ▼
         MMR 多样性排序（可选）
              │
              ▼
         检测多源冲突
              │
              ▼
         构建 RAG Prompt
              │
              ▼
         调用 LLM 生成答案
              │
              ▼
         返回答案 + 来源 + 置信度
```

### 3.3 表格回填流程

```
用户 ──▶ POST /fill
              │
              ▼
         识别模板字段
         ├── DOCX → 占位符格式 {{字段名}}
         └── XLSX → 直接用 openpyxl 读取表头列名（绕过 LLM）
              │
              ▼
         读取已索引的数据源
         ├── XLSX 源文件 → 保留 CSV 格式（LLM 能识别列标题）
         └── 15000 字符截断保护
              │
              ▼
         LLM 提取字段值
         ├── DOCX → 模糊匹配占位符
         └── XLSX → 精确匹配列名（避免误匹配）
              │
              ▼
         字段模糊匹配（处理表述差异）
         ├── 精确匹配(3分) > 包含(2分) > 子串(1分)
         └── 行溢出 → 取最后一行重复填充
              │
              ▼
         写入模板文件
         ├── DOCX → python-docx 替换
         └── XLSX → 智能追加行模式
             ├── 表头识别（选非空列最多的行）
             ├── 合并单元格处理（只读主格）
             ├── 写入防重（written_cells 追踪）
             └── 跨行合并跳过
              │
              ▼
         返回输出文件路径
```

## 四、核心配置参数汇总

### 4.1 模型配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `llm_api_key` | - | LLM API 密钥 |
| `llm_base_url` | - | LLM API 基础地址 |
| `llm_model` | Qwen/Qwen2.5-7B-Instruct | 对话模型 |

| `embed_model` | nomic-embed-text | 文本嵌入模型 |
| `reranker_model` | Qwen/Qwen3-Reranker-0.6B | CrossEncoder 重排模型 |

### 4.2 检索配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `vector_weight` | 0.6 | 向量检索在混合检索中的权重 |
| `bm25_weight` | 0.4 | BM25 检索在混合检索中的权重 |
| `top_k` | 8 | 初始检索返回的 chunk 数量 |
| `reranker_top_k` | 5 | CrossEncoder 重排后保留的 chunk 数 |
| `bm25s_method` | "BM25" | BM25S 算法变体 |
| `bm25s_k1` | 1.5 | BM25S k1 参数 |
| `bm25s_b` | 0.75 | BM25S b 参数 |

### 4.3 排序与融合配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `reranker_enabled` | True | 是否启用 CrossEncoder 重排 |
| `mmr_enabled` | True | 是否启用 MMR 多样性排序 |
| `mmr_lambda` | 0.7 | MMR λ 参数（1=完全相关，0=完全多样） |
| `fusion_method` | "rrf" | 结果融合方法（支持 rrf/score_sum） |
| `rrf_k` | 60 | RRF 融合超参数 |

### 4.4 缓存与存储配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `redis_host` | localhost | Redis 主机地址 |

| `redis_ttl` | 3600 | 缓存 TTL（秒） |
| `sqlite_path` | db/app.db | SQLite 数据库路径 |
| `chroma_path` | db/chroma | ChromaDB 持久化路径 |
